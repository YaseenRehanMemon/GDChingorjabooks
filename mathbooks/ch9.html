9.1 Linear Inequalities in One Variable

A linear inequality in one variable is a mathematical statement that uses inequality symbols (\(<\), \(>\), \(\le\), \(\ge\)) to compare a linear expression with a constant. Examples include \(x < -5\) or \(2x+4 \ge 10\).

9.1.1 Algebraic Solutions and Number Line Representation

To solve a linear inequality means to find all values of the variable that make the inequality true. The solution is often an interval or a set of numbers, which can be graphically represented on a number line.

Graphical Representation on a Number Line

  • For \(< \text{ or } >\), use an open circle or parenthesis to indicate that the endpoint is not included.
  • For \(\le \text{ or } \ge\), use a closed circle or bracket to indicate that the endpoint is included.

Example: The solution for \(x < 3\) is represented by an open circle at 3 and a line shaded to the left. In interval notation, this is \((-\infty, 3)\).

9.2 Linear Inequalities in Two Variables

A linear inequality in two variables, such as \(ax + by < c\), represents a region (a half-plane) on a two-dimensional coordinate plane. The solution set consists of all ordered pairs \((x, y)\) that satisfy the inequality.

9.2.1 Graphical Interpretation

To graph a linear inequality in two variables:

  1. Replace the inequality sign with an equality sign to get the equation of the boundary line (e.g., \(ax + by = c\)).
  2. Graph the boundary line. If the original inequality is \(<\) or \(>\), use a dashed line. If it's \(\le\) or \(\ge\), use a solid line.
  3. Choose a test point (e.g., \((0, 0)\) if it's not on the line) and substitute its coordinates into the original inequality.
  4. If the test point satisfies the inequality, shade the region containing the test point. Otherwise, shade the opposite region.

9.3 Linear Programming Problems

Linear programming (LP) is a mathematical technique used to find the best possible outcome (maximum or minimum value) of a linear objective function, subject to a set of linear inequalities called constraints.

9.3.1 Key Components of an LP Problem

  • Objective Function: The linear function to be maximized or minimized (e.g., \(Z = ax + by\)).
  • Decision Variables: The variables \((x, y)\) whose values need to be determined to optimize the objective function.
  • Constraints: The linear inequalities that limit the possible values of the decision variables. These often include non-negativity constraints like \(x \ge 0, y \ge 0\).
  • Feasible Region: The region on the graph that satisfies all the constraints simultaneously. The optimal solution always lies at one of the corner points (vertices) of this region.

9.3.2 Graphical Method for Solving LP Problems

  1. Formulate the LP problem by writing the objective function and all constraints.
  2. Graph all the constraint inequalities to identify the feasible region.
  3. Find the coordinates of all corner points (vertices) of the feasible region.
  4. Substitute the coordinates of each corner point into the objective function.
  5. The point that gives the maximum (or minimum) value for the objective function is the optimal solution.

Chapter Summary

  • Linear Inequalities: Mathematical statements using \(<, >, \le, \ge\).
  • One Variable: Solutions are intervals on a number line.
  • Two Variables: Solutions are shaded regions (half-planes) on a coordinate plane.
  • Linear Programming (LP): A method to optimize (maximize/minimize) a linear objective function subject to a set of linear constraints.
  • The feasible region is where all constraints are satisfied, and the optimal solution for an LP problem is found at one of its corner points.
